# Отчёт: Задолженность перед клиентами

**Версия:** 3.1  
**Последнее обновление:** 30 марта 2026  
**Автор:** ИТ-отдел Бризекс

## Что это такое

Автоматический отчёт, который показывает, **сколько мы должны клиентам** — товары, за которые клиенты заплатили, но ещё не получили (зарезервированные устройства).

**Google Таблица (основной доступ):** https://docs.google.com/spreadsheets/d/1GuFU4MaFG9KagIxVbeUW6QV2RuLtj_OebARltjoMrz0/edit  
**GitHub репозиторий:** https://github.com/Breezeks-cloud/moysklad-debt-report  
**Автообновление:** понедельник 09:00 МСК + пятница 06:00 МСК

---

## Листы отчёта (8 листов)

| Лист | Назначение |
|---|---|
| **Сводка** | Дашборд: ключевые цифры, разбивка по категориям, ТОП-10 должников, блок аномалий |
| **Обеспеченность CTM** | Сводка по заготовкам AIRNANNY: сколько должны устройств CTM, заготовок на складе, % обеспеченности, дефицит/профицит. Разбивка задолженности по конфигурациям |
| **Бризеры** | Иерархия: производитель → модель → конфигурация. Кол-во, долг, себестоимость + строка **ИТОГО** |
| **Товары (все)** | Все товары в резерве с группировкой по категориям |
| **Детализация** | Построчная привязка: клиент → заказ → товар → кол-во → долг → себестоимость (только активные) |
| `_Справочник` | Закупочные цены товаров (buyPrice / kitPrice) |
| `_API_Клиенты` | Исходные данные по клиентам из МойСклад (имя, код, тел., тип, баланс, долг, заказы, статус) |
| `_API_Позиции` | Исходные данные по позициям заказов (клиент, заказ, товар, кол-во, долг, категория) |

## Логика расчётов

### Кто попадает в отчёт

- Контрагенты с **положительным балансом** (мы должны им деньги/товар)
- Заказы с **01.01.2023** по текущую дату
- Исключены: Микроклиматика, ИП Гончаров, ООО Бризекс, тестовые (тест, ананас, четвёртый, ромашка)

### Долг клиента

```
Долг = payedSum − shippedSum  (из заказа покупателя, в рублях)
```

Распределяется по позициям **пропорционально дисконтированной цене** каждой позиции (цена после скидки).

### Статусы

| Статус | Описание |
|---|---|
| **Активный** | Есть хотя бы один открытый заказ с payed > shipped |
| **Закрытый** | Все заказы завершены или нет заказов с долгом. Лист «Закрытые с долгом» |

### Себестоимость

`buyPrice` из карточки товара × кол-во. Если `buyPrice` нет — берётся `kitPrice` (закупочная/минимальная цена из прайс-листа).

### Категории товаров

| Категория | Признак |
|---|---|
| **Бризер** | airnanny, tion, бризер, fanmaster, ballu, royal clima, turkov и др. (без фильтров/запчастей) |
| **Сплит/Кондиционер** | сплит, кондиционер, split, inverter |
| **Услуга** | `meta.type = "service"` |
| **Прочее** | Всё остальное (фильтры, запчасти, аксессуары, крепёж) |

### Обеспеченность CTM (лист «Обеспеченность CTM»)

Отслеживает возможность покрыть задолженность по бризерам **AIRNANNY** (CTM) за счёт **заготовок** на складе.

```
Обеспеченность = min(заготовки_на_складе / должны_устройств_CTM, 100%)
Дефицит = max(0, должны − заготовки_на_складе)
```

**Допущения:**
- Заготовки универсальны → из одной заготовки собирается **любая** конфигурация AIRNANNY A7
- Детали для сборки конфигурации (фильтры, плата и т.д.) всегда доступны
- Партнёрские бризеры (ПТМ: Tion, Ballu и др.) в расчёт **не включаются** — их можно быстро закупить

**Источник данных по остаткам:** `GET /report/stock/all` МойСклад, ключевые слова: «заготовка airnanny», «заготовка a7»

## Быстрый старт

### Обновить отчёт вручную (через Cursor AI)

Напишите в чат:
> «Обнови отчёт по задолженности перед клиентами»

ИИ-агент запустит GitHub Actions workflow, который:
1. Загрузит данные из МойСклад API (~15 мин)
2. Сформирует xlsx-файл (артефакт)
3. Обновит Google Таблицу

### Запустить workflow вручную

GitHub → [Actions](https://github.com/Breezeks-cloud/moysklad-debt-report/actions) → «Weekly Report Update» → Run workflow

### Запустить скрипты локально

```bash
cd "scripts/"
export MOYSKLAD_TOKEN="ваш-токен"
export SPREADSHEET_ID="1GuFU4MaFG9KagIxVbeUW6QV2RuLtj_OebARltjoMrz0"

python3 01_fetch_data.py         # Сбор данных (~15 мин)
python3 02_build_report.py       # Генерация xlsx (~10 сек)
python3 03_upload_gsheets.py     # Выгрузка в Google Sheets (~30 сек)
```

## Конфигурация

### GitHub Secrets (репозиторий `Breezeks-cloud/moysklad-debt-report`)

| Секрет | Назначение |
|---|---|
| `MOYSKLAD_TOKEN` | API-токен МойСклад (Bearer) |
| `SPREADSHEET_ID` | ID Google Таблицы (`1GuFU4MaFG9KagIxVbeUW6QV2RuLtj_OebARltjoMrz0`) |
| `GOOGLE_CREDENTIALS_JSON` | Service Account JSON для Google Sheets API |

> **Токены НЕ хранятся в коде.** Только через переменные окружения или GitHub Secrets.

### Расписание автообновления

| День | Время (МСК) | Cron |
|---|---|---|
| Понедельник | 09:00 | `0 6 * * 1` (UTC) |
| Пятница | 06:00 | `0 3 * * 5` (UTC) |
