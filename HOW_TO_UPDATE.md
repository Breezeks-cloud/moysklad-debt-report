# Инструкция по обновлению отчёта

## Способ 1 — Через Cursor AI (рекомендуется)

Просто напишите в чат Cursor:

> **"Обнови отчёт по задолженности перед клиентами"**

Агент сам:
1. Запустит сбор данных из МойСклад
2. Сформирует новый xlsx
3. Сохранит файл в `0_Inbox/`
4. Сообщит итоговые цифры

---

## Способ 2 — Запуск скрипта вручную

### Требования

- Python 3.8+
- `pip install openpyxl`

### Шаги

```bash
# 1. Перейти в папку скриптов
cd "/Users/m_goncharov/Nextcloud/0_BREEZEKS/3. РЕСУРСЫ/391-БИБЛИОТЕКА/База знаний по МойСклад/Отчёт — Задолженность перед клиентами/scripts"

# 2. Собрать данные из API (~6-8 минут)
python3 01_fetch_data.py

# 3. Сформировать Excel (~10 секунд)
python3 02_build_report.py
```

Файл будет сохранён в `~/Nextcloud/0_Inbox/Задолженность_перед_клиентами_v2.xlsx`.

### Только обновить Excel (без повторного запроса API)

Если данные уже загружены (pkl-кэш свежий):

```bash
python3 02_build_report.py
```

### Выгрузить в Google Sheets

После настройки (см. [GOOGLE_SHEETS_SETUP.md](./GOOGLE_SHEETS_SETUP.md)):

```bash
export SPREADSHEET_ID="ваш-id-таблицы"
python3 03_upload_gsheets.py
```

Или полный цикл:

```bash
python3 01_fetch_data.py && python3 03_upload_gsheets.py
```

## Способ 3 — Автоматически через GitHub Actions

Настройте репозиторий — и отчёт будет обновляться **каждый понедельник в 09:00**.

### Первоначальная настройка

1. Создайте репозиторий на GitHub (приватный)
2. Добавьте файлы из папки `scripts/` и `.github/`
3. Добавьте секрет `MOYSKLAD_TOKEN` в настройках репозитория:
   - Settings → Secrets and variables → Actions → New repository secret
   - Имя: `MOYSKLAD_TOKEN`
   - Значение: `5cb4bc34b5eb0ba857c61a016fca8c223922ae40`
4. Настройте загрузку файла в Nextcloud (см. ниже)

### Получение файла из GitHub Actions

После каждого запуска workflow файл xlsx загружается как **Artifact**. Скачать можно:
- Вручную через интерфейс GitHub: Actions → последний запуск → Artifacts
- Автоматически через Nextcloud WebDAV (см. `scripts/03_upload_nextcloud.py`)

### Запустить обновление вручную (без ожидания расписания)

GitHub → Actions → "Weekly Report Update" → Run workflow

## Что делать, если что-то пошло не так

### Ошибка: "HTTP Error 415"
Устаревшая версия скрипта. Обновите до актуальной из папки `scripts/`.

### Ошибка: "HTTP Error 401"
Истёк или невалидный токен МойСклад. Обновите токен в скрипте или в GitHub Secrets.

### Пустой отчёт / мало строк
Проверьте: МойСклад доступен? Попробуйте запустить `01_fetch_data.py` с флагом `--debug`.

### Неверные цифры по себестоимости
Откройте лист `_Справочник` в xlsx. Проверьте столбцы `buyPrice` и `kitPrice`. При необходимости скорректируйте вручную столбец `Себест. за ед.` (он не перезаписывается при обновлении данных).

---

## Что НЕ нужно делать

- ❌ Не редактируйте листы `_API_*` — они перезаписываются при каждом обновлении
- ❌ Не меняйте имена листов — формулы ссылаются на конкретные названия
- ✅ Правки цен можно вносить в `_Справочник`, столбец G (`Себест. за ед.`)

## Частота обновления

| Способ | Частота |
|---|---|
| Cursor AI (вручную) | По запросу |
| Скрипт вручную | По запросу |
| GitHub Actions | Каждый понедельник 09:00 МСК |

### Что обновляется при каждом запуске

```
МойСклад API
      ↓
01_fetch_data.py    (~7 мин)   — свежие данные
      ↓
02_build_report.py  (~10 сек)  — обновляет xlsx в Nextcloud
      ↓
03_upload_gsheets.py (~30 сек) — обновляет Google Таблицу
      ↓
Павел открывает ссылку и видит актуальные данные
```

**Рекомендация:** таблица обновляется в начале и в конце рабочей недели — перед понедельничным совещанием и перед выходными.
